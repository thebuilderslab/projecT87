1.2 Refactor connectWallet to await auth before loading data

Show the current connectWallet() / authenticateWallet() implementations.

Refactor them so that:

Step 1: user connects via wallet provider and you get the address.

Step 2: you call the backend auth endpoint, await the response, and only after storing authToken and userId in state and localStorage do you fire dashboard data fetches.

Example:

javascript
async function connectWallet() {
  try {
    const address = await requestWalletAddress();
    state.walletAddress = address;

    // Step 2: authenticate and await
    const auth = await authenticateWallet(address);
    state.authToken = auth.token;
    state.userId = auth.userId;
    localStorage.setItem('rea_wallet_session', JSON.stringify({
      walletAddress: address,
      authToken: auth.token,
      userId: auth.userId,
    }));

    // Step 3: only now load dashboard data (in parallel if you want)
    await Promise.all([
      loadDefiPanel(),
      loadTowns(),
      loadRecentPayouts(),
      // any other initial loaders
    ]);

  } catch (e) {
    console.error('connectWallet failed', e);
    fullDisconnect();
  }
}
