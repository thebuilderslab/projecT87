PnL Control: All operational parameters (Growth, Capacity, Debt Swap) must be adjustable from the dashboard by inputting a Target PnL amount, not just a dollar threshold.

Full Synchronization: The UI and backend must stay in real-time sync using a dedicated channel (WebSockets) to reflect parameter changes and live metric updates.

Stability: Immediate resolution of the API rate-limiting issue is required for full operational status.

This plan details the construction of the dashboard to meet the adjusted PnL control and real-time synchronization requirements.

2. PnL Linkage & Conversion Logic
The backend must act as the authority for converting PnL targets into operational thresholds.

Conversion Handler: Create a dedicated Python module (pnl_converter.py) that handles the logic:

Operational USD Threshold=f(Target PnL, Market Conditions, USD/PnL Rate)
Agent Logic Update: The Autonomous Agent must read only the calculated USD threshold from the config.json after the backend has converted and saved the user's Target PnL. This ensures the agent never sees a raw PnL figure, only the actionable USD threshold.

3. Backend API and Synchronization (WebSockets)
To ensure both the UI and agent are always in sync, a dual-channel communication structure is mandatory.

REST API for Writes (Secure):

Endpoint: PUT /api/config/{parameter_name}

Action: The dashboard sends a PnL value (e.g., pnl_growth_target: 7.50) to this secure, authenticated endpoint. The backend validates the input, runs the PnL-to-USD conversion, updates the config.json, and then triggers the WebSocket broadcast.

WebSocket for Reads (Real-Time Sync):

Endpoint: ws://[host]/ws/dashboard

Action: Establish a persistent WebSocket connection upon dashboard load.

Broadcast: The backend must immediately broadcast the full system state (new config.json parameters, live Health Factor, Available Borrows, and Trigger Status) every time a parameter is saved or an autonomous action is executed.

5. Synchronization Workflow
User Action: User enters $7.50 into the "Target Growth PnL" field and clicks Save.

Dashboard Action: Dashboard sends a PUT /api/config/growth request with the new PnL value.

Backend Action: The backend receives the request, runs the PnL-to-USD conversion, updates config.json with the new operational USD threshold (e.g., growth_threshold_usd: 72.50), and immediately sends the entire new state via WebSocket broadcast.

Real-Time Sync: The dashboard's WebSocket listener receives the broadcast and instantly updates the "Target Growth PnL" field to $7.50 and the "Operational Collateral Growth" metric to $72.50, ensuring the UI/backend state is fully synchronized.
