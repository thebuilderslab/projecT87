// Check existing credit delegations first
const creditDelegationCalls = [];
for (const [symbol, debtTokenAddr] of 
     Object.entries(VARIABLE_DEBT_TOKENS)) {
  const debtToken = new ethers.Contract(
    debtTokenAddr, DEBT_TOKEN_ABI, signer
  );
  const allowance = await debtToken
    .borrowAllowance(userWallet, 
                     DELEGATION_MANAGER);
  if (allowance.eq(0)) {

      token: symbol,
      contract: debtToken

if (creditDelegationCalls.length > 0) {

}

## Permission Layers for Full Automation

Three separate on-chain permission layers 
are required before the bot can execute 
strategies on behalf of a user wallet:

Layer 1: DelegationManager flags
  Contract: REAADelegationManager
  Function: approveDelegation(...)
  Flags: allowSupply, allowBorrow, 
         allowRepay, allowWithdraw = true
  Signed by: User

Layer 2: ERC20 token approvals
  Tokens: DAI, WETH, WBTC, USDC, USDT
  Spenders: DelegationManager, Aave Pool, 
            Uniswap Router
  Amount: MaxUint256
  Signed by: User (via Multicall3)

Layer 3: Aave V3 credit delegation
  Contracts: Variable debt tokens for each 
             supported asset
  Function: approveDelegation(
    DelegationManager, MaxUint256)
  Required for: Any borrow on behalf of user
  Signed by: User (via Multicall3)
  
  Variable debt token addresses (Arbitrum):
    DAI:  0x8619d80FB014...
    WBTC: 0x92b42c668400...
    WETH: 0x0c84331e39d6...
    USDC: 0xFCCf3cAbbe80...
    USDT: 0x6ab707Aca953...
