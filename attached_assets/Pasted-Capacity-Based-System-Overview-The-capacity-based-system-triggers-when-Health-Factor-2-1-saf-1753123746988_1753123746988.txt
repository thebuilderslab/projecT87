Capacity-Based System Overview
The capacity-based system triggers when:

Health Factor > 2.1 (safe threshold)
Available Borrows ≥ $40 (sufficient capacity)
Capacity Utilization < 20% (underutilized position)
Complete Sequence Flow
1. System Activation Check
capacity_utilization = available_borrows / max(total_collateral * 0.8, 1)
capacity_conditions_met = (
    health_factor > 2.1 and
    available_borrows >= 40.0 and
    capacity_utilization < 0.20
)
2. DAI Borrowing Phase
Calculates optimal borrow amount (typically 50-80% of available capacity)
Borrows DAI from Aave using available borrowing power
Waits for transaction confirmation
3. DAI Allocation Strategy
The borrowed DAI gets allocated across three operations:

allocation = {
    'wbtc_swap': borrowed_amount * 0.4,    # 40% for WBTC
    'weth_swap': borrowed_amount * 0.3,    # 30% for WETH  
    'direct_supply': borrowed_amount * 0.3  # 30% direct DAI supply
}
4. Swap and Supply Sequence
Step 1: DAI → WBTC → Supply

Swaps 40% of DAI for WBTC via Uniswap V3 (0.05% fee tier)
Supplies received WBTC to Aave as collateral
Increases collateral diversity and value
Step 2: DAI → WETH → Supply

Swaps 30% of DAI for WETH via Uniswap V3 (0.05% fee tier)
Supplies received WETH to Aave as collateral
Further diversifies collateral portfolio
Step 3: Direct DAI Supply

Supplies remaining 30% DAI directly to Aave
Maintains stablecoin exposure in portfolio
5. Technical Implementation
Each swap operation follows this pattern:

# Example: DAI to WBTC swap
swap_result = self.uniswap.swap_tokens(
    self.dai_address,      # Input token
    self.wbtc_address,     # Output token  
    allocation['wbtc_swap'], # Amount in USD
    500                    # 0.05% fee tier
)
if swap_result:
    # Wait for swap confirmation
    time.sleep(10)
    
    # Get received WBTC balance
    wbtc_balance = self.aave.get_token_balance(self.wbtc_address)
    
    # Supply WBTC to Aave
    supply_result = self.aave.supply_wbtc_to_aave(wbtc_balance)
6. Benefits of This System
Leveraged Growth: Uses borrowed funds to acquire more collateral
Diversification: Spreads risk across WBTC, WETH, and DAI
Yield Optimization: All supplied assets earn Aave lending yields
Capital Efficiency: Maximizes utilization of borrowing capacity
7. Safety Mechanisms
Maintains health factor > 2.1 throughout operations
Uses 30% gas price buffer for reliable transaction execution
Includes comprehensive error handling and rollback procedures
Monitors for emergency stop conditions
This capacity-based approach transforms idle borrowing power into productive, yield-generating collateral while maintaining conservative risk management through diversified asset allocation.
