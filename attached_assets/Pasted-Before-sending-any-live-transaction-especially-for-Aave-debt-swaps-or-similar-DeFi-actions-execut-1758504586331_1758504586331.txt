Before sending any live transaction (especially for Aave debt swaps or similar DeFi actions), execute the following steps to minimize the risk of transaction revert:

1. **Parameter Validation:**  
   - Validate all required input parameters (amounts, addresses, asset types) for correctness and protocol limits before building the transaction.
   - Check for zero, negative, or out-of-range values.

2. **Pre-Transaction Simulation:**  
   - Always perform an off-chain simulation/dry-run of the transaction using `eth_call`, Tenderly, or the protocol's `staticCall/callStatic` method.
   - Abort submission and report the revert reason if the simulation fails.

3. **Balance and Allowance Checks:**  
   - Verify that the sender wallet has sufficient balance of the required asset and enough ETH for transaction fees.
   - Check ERC-20 token allowances and prompt for approval if insufficient.

4. **Health Factor/Post-Action State:**  
   - For borrowing/lending, estimate and confirm that the user's health factor and collateralization will remain above the safe threshold after the swap.
   - Abort or warn if the post-operation state will risk liquidation or breach protocol rules.

5. **Network Fee (Gas) Settings:**  
   - Set both gas price and gas limit according to the current Arbitrum network recommendations.
   - Increase gas limit for complex transactions, and never use outdated fee settings.

6. **Live Contract & ABI Checks:**  
   - Double-check the contract address and ABI version against the official Aave (or protocol) documentation before submitting.

7. **Error Handling and Reporting:**  
   - If a revert occurs, capture and output the full revert reason and all parameters.
   - Add this information to the log for debugging and future prevention.

After implementing these steps, summarize the process in a report before final submission, and only send the real transaction if every pre-check passes. If any check fails, output structured diagnostics and halt action.
