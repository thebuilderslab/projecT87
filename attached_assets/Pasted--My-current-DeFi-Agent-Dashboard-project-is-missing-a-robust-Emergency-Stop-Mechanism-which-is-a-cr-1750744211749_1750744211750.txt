"My current DeFi Agent Dashboard project is missing a robust Emergency Stop Mechanism, which is a critical, non-negotiable component for mainnet deployment and safe real-money operation.

Goal: Implement a foundational, off-chain emergency stop mechanism that allows for immediate cessation of automated agent operations.

Instructions:

Agent-Side (Backend - agent.py):

Stop Flag: Implement a system where agent.py will immediately cease its main operational loop if a specific stop_flag is set to True.
Mechanism: This stop_flag should be read from an external, easily modifiable source.
Primary Method (File-based): The agent should check for the presence of a designated file (e.g., EMERGENCY_STOP_ACTIVE.flag) at the beginning of every iteration of its main loop. If this file exists, the agent must exit its loop cleanly, ceasing all further Aave interactions.
Fallback/Alternative (Configuration Variable): As a secondary method, or if a file-based approach isn't feasible in the environment, consider checking a specific environment variable or a configuration value that can be toggled.
Graceful Exit: Upon detecting the stop_flag, the agent should log a message indicating the emergency stop was activated and then terminate gracefully. It should not attempt to initiate any new transactions.
Prevent New Transactions: Ensure the stop_flag check is performed before any calls that initiate on-chain transactions (e.g., deposit, borrow, repay).
Dashboard-Side (Frontend - React/HTML):

Emergency Stop Button: Add a prominent, clearly labeled "EMERGENCY STOP" button to the "Manual Controls" section of the dashboard.
Button Action: When pressed, this button should:
Trigger a backend endpoint (e.g., /api/activate_emergency_stop) to set the stop_flag (e.g., create the EMERGENCY_STOP_ACTIVE.flag file).
Display a confirmation message to the user on the dashboard that the emergency stop has been activated.
Status Indicator: Implement a visual indicator on the dashboard (e.g., a red banner, "AGENT STOPPED" text) that updates in real-time to reflect whether the emergency stop is currently active. This indicator should read the state from the backend.
Reset Mechanism:

Separate Action: Implement a separate action or button (e.g., "Reset Agent" or "Resume Operations") that, when triggered, clears the stop_flag (e.g., deletes EMERGENCY_STOP_ACTIVE.flag). This action should require a deliberate confirmation from the user (e.g., a modal with "Are you sure you want to resume?").
Restart Agent: Clearly instruct the user that after resetting the flag, they may need to manually restart the agent.py script in Replit for operations to resume.
Logging:

Ensure that the agent logs when the emergency stop is activated and when it is deactivated/reset, including timestamps.
Output Expected:

Code Snippets: Provide the necessary Python code changes for agent.py (stop flag check, graceful exit logic, logging) and the relevant frontend code (button, API call, status indicator).
API Endpoint: Define the new API endpoint(s) for activating and deactivating the stop.
Instructions: Provide clear, concise instructions on how to use the emergency stop from the dashboard and how to reset/resume operations."
