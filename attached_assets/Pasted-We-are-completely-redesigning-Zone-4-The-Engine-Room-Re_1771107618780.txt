We are completely redesigning Zone 4 (The Engine Room). Replace the standard meters with a Personified Video Avatar that reacts to the system state.

1. The New Layout (Visual Structure):

Container: Zone 4 is now a "Video Card."

Background: A full-width, auto-playing HTML5 <video> element.

The "HUD" (Heads-Up Display): Compress the original data (Timers, Capacity Bar, Yield Farm Button) into a semi-transparent, high-tech overlay panel at the bottom 25% of the card.

Style: Think "Iron Man HUD" – minimalist text, thin progress bars, clear green/red status lights overlaid on the video.

2. Video State Logic (CRITICAL):
Create a updateAvatarState(status) function that swaps the <video src> based on live triggers.

Master Rule: The system MUST always default back to the IDLE state after any other video finishes playing.

3. State Definitions & Behaviors:

STATE A: IDLE (The Default Loop)

Trigger: System is monitoring (HF > Thresholds) but no action is occurring.

Video File: assetsavatar_idle.mp4

Behavior: Continuous Infinite Loop. This video must replay over and over again without stopping until a specific event trigger (Growth, Shield, or Harvest) interrupts it.

STATE B: GROWTH SURGE

Trigger: last_action == "Growth" OR Growth Cooldown just started.

Video File: assetsavatar_growth.mp4

Behavior: Play ONCE, then immediately revert to STATE A (IDLE).

STATE C: DEFENSIVE SHIELD

Trigger: last_action == "Macro/Micro Short" OR Liability Cooldown just started.

Video File: assetsavatar_shield.mp4

Behavior: Play ONCE, then immediately revert to STATE A (IDLE).

STATE D: HARVEST (Success)

Trigger: User clicks "CLAIM USDC" button.

Video File: assetsavatar_harvest.mp4

Behavior: Play ONCE, then immediately revert to STATE A (IDLE).

4. The "Smart Text" Overlay:
Inside the HUD, replace the generic title with the "Smart Status" text:

If Idle: "👁️ SENTINEL: SCANNING MARKET..."

If Cooling: "🔋 RECHARGING: [MM:SS]"

If Paused: "⛔ HOLDING POSITION: LOW HF"

5. Implementation Details:

Use object-fit: cover for the video to fill the card.

Add an event listener: video.onended = () => { if (currentVideo !== idle) playIdle(); } to ensure the automatic return to the loop.

Ensure transitions are smooth (no black flashes).

Add muted and playsinline attributes.
